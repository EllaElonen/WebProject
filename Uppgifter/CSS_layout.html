Page Layout

Why HTML Tables are not optimal

Previously, before CSS existed, HTML tables were used for design purposes. For following reasons, HTML tables are not optimal compared to CSS, and should be restricted to tables that are needed for content purposes:

  • They use unnecessary bytes and time
  • They can divide images and prevent copying of text
  • Tables describe the presentation, not the contents, therefore they should be handled by CSS
  • They are not easy to handle for screen readers and search engines
  • They make redesign hard; CSS properties are easier to change

CSS Layout

Things you might have to consider in designing with CSS is screen resolution, browser, window size, hardware etc. CSS has three layout options: fixed, fluid and elastic.

Fluid design

In fluid (liquid) layout, most components have percentage widths, which means they adapt to the users screen. Some components might still have fixed widths. This layout might be a bit hard to customize, and you may need to set min and max layouts. You can, however, optimize the use of screen. This design is normally the most user-friendly, because it adapts to the users conditions. With very large resolutions, it can create extra space, though.

An important concept here is the max-width property of CSS. It decides the maximum width an element can get. For example, paragraph lines are shortened in lack of space, like in this paragraph.

The opposite of max-width is min-width, which is illustrated by this paragraph. Display: inline-block allows you to set the width and height.

Other things important to think about is to avoid using complicated graphics and hard techniques, that may make the page hard to maintain.

Fixed Width Design

Fixed width has the advantage that everybody will se the same width of things, they will not move or change size. Some of the the cons are exessive white space, the need of horizontal scroll bars, and lower overall usability. When you build a page with fixed design, it is important to center your wrapper div (on this page, the article tag is used because semantic tags are preferrable in HTML5).

Elastic Width Design

Elastic Width design mixes the two previous ways of designing. It is many times preferred by designers. It works by defining widths with em - a square of the font size. It adapts to the users text-size. It is hard to get it right, it can take a lot of skills and testing, and extra style-sheets might be needed.
