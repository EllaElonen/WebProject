The Structure of an HTML Page

When a browser loads a webpage, it creates a Document Object Model, DOM, of the page. You can use JavaScript to change objects and their presentation styles in this structure. For example, you could replace a paragraph, a picture or a table with another, or give it a background color as a response to a click from a user, for example. So far, however, we are only concerned with the HTML-part.

The basic structure of an HTML-page consists of a <!DOCTYPE>-which defines the markup language so that the browser can display content correctly, and two >html<-tags, to tell the browser that the page is written in HTML. containing a head and a body element. The head contains metainformation about the page, and can also contain links to JavaScript and CSS files

Meta information in the head element

The meta tags contained in the head element of a page can contain meta information about the page, like encoding, author, description, keywords, page title, software that was used to generate it, how search engines should treat it, how often the browser should refresh it, etc. Here, for example, <meta http-equiv="refresh" content="30"> was used to tell the page to refresh every 30 seconds.

This page contains the following meta tags:
charset - specifies the encoding, for example utf-8
author - the person who created it
description - a description of the content
generator - The software used to generate it. Here, the value is set to zero because no software was used for generating the document.
keywords - what the search engines should be looking for
robots - defining desired behavior for search engines (set to noindex, nofollow that tells robots
not to index it or follow links on that page).

Span

Marking text can be done with the span element. If you want to make something apply to certain parts of the text, you surround them with span tags. This can be done for styling or semantic purposes. This sentence is surrounded with span tags with a class attribute illustrateSpan.

Quotes

Block quotes can be written in blockquote tags. This illustrates a block quote from another website, https://www.quackit.com/html/tags/html_blockquote_tag.cfm:

The HTML <blockquote> tag is used for creating the 'blockquote' element. This element represents content that is quoted from another source. Browsers usually render <blockquote> text as indented text. This results in an indented paragraph (or multiple indented paragraphs if the <blockquote> spans multiple paragraphs). For shorter quoted text that needs to display within a non-quoted paragraph, use <q> tag. Most browsers surround <q> text with quotation marks. Syntax The <blockquote> tag is written as <blockquote></blockquote> with the quoted content inserted between the start and end tag. Like this: <blockquote>Quoted text here...<blockquote></blockquote> - quackit

The div element

The div element can be used as a container for basically any kind of content, like text, ictures, lists, etc. In HTML5, semantic elements like article and section are preferred when applicable, but otherwise it is still okay to use div as a container when needed.

This is an exmple of a div element. When filled with text, it can look like a regular paragraph, if nothing else has been specified. You could specify different classes of div elements, or give them an id, and apply a certain kind of styling to them if you wish.

Tables

Tables are written by writing text inside td tags for regular rows, and <th> for headings. Every piece of data inside the <tr> tags is surrounded by a <td>. Background color is added to columns by enclosing them in a <colgroup span="2"> tag. The color is defined in a separate CSS file. Read more about how to style with CSS here Here.
