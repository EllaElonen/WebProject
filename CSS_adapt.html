CSS Layouts: How to make your document suitable for printing

@page and margin @ rules

This page is an example of a printable layout. In order to make that, an important concept is @page rule. It lets you control the size and orientation of the page, margins, borders and padding, headers and footers, page counters and inheritance issues. For example, this page sets the margin for all pages like this: 

					@page {
						margin: 10cm;
					}

You can also define how to crop the pages, and the bleed area - the area outside the page box. All of this will be applied to all pages. You can also define some pages as horizontal and some as vertical, and then apply orientation or formatting just to the vertical ones, for example. 

					section{
						page vertical;
						}
				
				

					@page vertical{

						size: A4 portrait;
					}

You could also use a pseudo class, for example: @page :blank to choose all the blank pages.

Margin @ rules identify one or many corners of a document, for example: @top-left-corner could be given a certain margin.

Page-breaks can be defined in different ways, but it is essential that the document know about where to insert a page-break. One example, used on this page:

Page breaks

					footer {
						page-break-after: always;

Page size

This page defines the size of the document body page to A4 portrait. If it included tables or pictures that would need to be printed horizontally, those pages would be given the value landscape. The size can also be set in pixels or inches, or as auto.

Orphans

You can also define orphans for your pages: The amount of minimum lines for an element that should be visible at the bottom of a page. For this document, it is set to two. The same goes for widows, only it refers to the top of a page.
